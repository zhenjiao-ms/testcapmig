# Metadata Definitions
`<!# Metadata Definitions API-->`

When analyzing the JSON for anything in the CAPS API, it can be very useful to have information about all the possible fields in the entities, including the optional ones that may or may not appear, depending on the state of the item. Luckily, there is an API to fetch metadata about this metadata. This is a rich source of understanding about the usage of a given field for an entity.

To get the metadata for the fields of a given entity type use the URI route:

`GET` _baseUri_`/v1/Organizations/`_orgId_`/MetadataDefinitions/`_entityTypes_

Parameter | Description
--- | ---
_baseUri_     | Base CAPS API URI.
_orgId_       | Id for your organization (environment).
_entityTypes_ | Comma-separated list of entity types to query. One or more of (`Article`, `Token`, `MediaContent`, `Outline`, `Handoff`, `Handback`, `DocSet`, `Query`), and so on.

## Sample Request

`GET http://caps-api-devint.azurewebsites.net/Organizations/694eacb19e664adc83d36c50cb0c996d/MetadataDefinitions/Article`
